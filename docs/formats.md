# SASS-BEM - Форматы

Многие миксины и функции в sass-bem принимают на вход данные определенных форматов, которые также могут быть описаны в документации как тип данных. Один и тот же формат может использоваться неоднократно в разных миксинах и функциях. Ниже представлено описание всех таких форматов.

## Формат `Entity`

Хэш, содержащий описание БЭМ-сущности. Как правило, используется внутри библиотеки, а все внешние миксины и функции принимают `FlexEntity`, который позже преобразуется к этому формату.

Выглядит следующим образом: `(block: a, elem: b, mod: c, val: d)`. Все поля опциональны.

## Формат `FlexEntity`

Предназначен для описания блока, элемента и модификатора разными способами, чтобы можно было использовать наиболее удобный и подходящий под ситуацию способ.

Любое поле, предназначенное для блока, может также быть списком, содержащим блок и элемент. Аналогично, поле для имени модификатора может быть списком, содержащим имя и значение модификатора. Однако, явный элемент и значение модификатора перезапишут аналогичные данные в списках.

Далее представлены допустимые данные для формата `FlexEntity`. Для примера используются блок, элемент, имя и значение модификатора с именами `a`, `b`, `c`, `d` соответственно.

**map (clean-entity)**: То же, что и `Entity`.

**map (short)**: `(b: a, e: b, m: c, v: d)`.

**map (verbose)**: `(block: a, element: b, modifier: c, value: d)`.

**list (entity)**: `(a, b)`. Также можно использовать список с одним элементом - он посчитается блоком.

**list (full)**: `(a, b, c, d)`.

**list (full-lists)**: `((a, b), (c, d))`. Чтобы первый элемент списка считался блоком, а второй - модификатором, нужно, чтобы хотя бы один элемент был списком.

**string (block)**: `a`.

## Формат `Mod`

Служит для описания модификатора. Может быть представлен как простой строкой (булевый модификатор), так и списком, где первый и второй элементы - ключ и значение соответственно.
